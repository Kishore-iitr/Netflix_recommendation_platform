## Overview
This repository implements an interactive **Streamlit** based recommendation system built **exclusively on the Netflix Prize dataset**. Users can:
- Explore the data (distribution of ratings, most‑rated movies).
- Choose what **fraction of the whole dataset** to use for training/testing (e.g., 10 % for quick experiments up to 100 % for full‑scale runs).
- Adjust the **train/test split proportion**.
- Select which models to train:
  - SVD (Optimized, optionally tuned with Optuna)
  - Item‑based KNN
  - ALS baseline
  - XGBoost Regressor
  - Random Forest Regressor
- Visualise evaluation metrics (RMSE, MAP@10) for the chosen models.
- Generate personalized recommendations for existing users or cold‑start new users.
- Get **LLM‑driven explanations** for the recommendations via OpenRouter.

## Setup
1. **Python 3.10+** is required.
2. Install dependencies (run in the project root):
   ```bash
   python -m venv venv
   .\\venv\\Scripts\\activate   # Windows
   pip install -r requirements.txt
   ```
   The `requirements.txt` (generated by `pip freeze`) includes:
   - streamlit
   - pandas, numpy, matplotlib, seaborn
   - scikit‑surprise
   - optuna
   - xgboost
   - scikit‑learn
   - requests
3. **Prepare the Netflix data**:
   - Place the original `combined_data_1.txt` inside `data/`.
   - Run the preprocessing script to generate a diverse sample:
     ```bash
     python preprocess.py
     ```
   - This creates `data/sampled_ratings.csv` (default 8 % movie sampling, users with ≥ 8 ratings).
   - The script can be re‑run with custom `sample_prob` and `min_ratings_per_user` arguments if you want a different density.

## Running the App
```bash
streamlit run app.py
```
Or double‑click `run.bat`.

## Using the UI
1. **Experiment Settings (sidebar)**
   - **Data Fraction** – Slider to pick what portion of the pre‑processed data to actually feed into the models (0.1 = 10 % of sampled rows, up to 1.0 = all rows).
   - **Test Set Size** – Fraction for the train/test split (default 0.2).
2. **Model Evaluation**
   - Choose one or more models.
   - Optionally enable Optuna for SVD hyper‑parameter search and set the number of trials.
   - Click *Run Model Training & Evaluation*.
   - Results table and bar charts for RMSE and MAP@10 appear.
3. **Personalized Recommendations**
   - **Existing User** – Pick a user ID and a trained model to see top‑10 recommendations.
   - **New User (Cold‑Start)** – Select favourite movies; the Item‑based KNN model instantly suggests similar titles.
4. **LLM Explanation**
   - Provide an OpenRouter API key, pick a model, and click *Generate Explanation* to receive a natural‑language rationale for the recommendation list.

## Extending / Customising
- **Different data split** – Adjust the `sample_fraction` or `test_size` sliders.
- **Add more models** – Extend `available_models` in `app.py` and implement them in `recommender.py`.
- **Change preprocessing** – Modify `preprocess_data` arguments for sampling probability, minimum user ratings, or add additional filters.
